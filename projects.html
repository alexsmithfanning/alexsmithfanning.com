The "It's Complicated" Projectclose

March 23rd, 2017

So dubbed for now as It's Complicated, due to the difficulty in explaining exactly what the product does without using two separate paragraphs of text.

So what does it do?

Basically it generates Android assets for use later on by an app. It intelligently resizes and configures everything on its own so developers don't have to worry about it. It also improves performance by a decent amount because it eliminates the need to generate assets on the fly as per the normal method. It does this by generating assets for many screen sizes simultaneously, which has the side effect of increasing app size by about fifteen percent.

Should I as a normal user care?

Nope! The only reason this project exists is for developers. It serves no purpose to the general public. The only way you could tell it was there was if it wasn't doing its job correctly.

I'm a developer; can you give me more details?

I can indeed! Normally the /res folder holds all the important information such as styles and drawables. Without my generator, you'd have to use Android's XML SVG format which sometimes isn't feasible. Generating everything in .png requires the long, laborious task of using Android Studio to convert drawable into the right sizes individually. With the generator though, it takes a standard SVG (or optionally a directory containing SVGs) and automatically exports them for the needed screen sizes. Although unlike the conventional Android Studio method, it can generate assets for many more screen sizes than normally possible by using XML tricks to avoid on-the-fly generation of assets at all.

If we go even more in depth, basically all the generator exists to do is wrap around other programs and give them a simpler, more collaborative interface. One of the biggest programs being used, inkscape, is a SVG editor with a GUI by day, and a powerhouse manipulation tool with a magnificent command line interface by night. Another main component is optipng, which does exactly what it says it does. It uses various compression tricks to reduce the size of lossless .png images without losing any quality. The program calls inkscape to render the .png files for many different screen sizes, and you can pass an option to my generator to also optimize the resulting assets with optipng.

Android's visual assets are stored in /res/drawable/ within the source code and compiled versions of the application. There's also DPI specifc folders for drawables within the /res folder: drawable-xxxhdpi, drawable-xxhdpi, drawable-xhdpi, drawable-hdpi, drawable-mdpi, and drawable-ldpi. DPI specifc drawables are never usually scalable, and are not generated on-the-fly. This makes them much faster to render, but much more difficult to get right as it can't be corrected in code without it looking terrible. These are the key to my generator. It takes what would normally be in /res/drawable/ as a scalable .xml and instead renders it with the specific DPI options as a non-scalable lossless .png image.
